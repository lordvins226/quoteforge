Every card and deck is a JSON document validated by Zod. Files are auto-detected by their `type` field: `"deck"` → carousel, `"card"` (or no `type`) → single card.

## Card shape

```json
{
  "type": "card",
  "template": "manifesto",
  "theme": "terminal-green",
  "size": "instagram-sq",
  "blocks": [
    {
      "id": "h",
      "type": "headline",
      "parts": [
        { "text": "Ship typographic cards ", "style": "normal" },
        { "text": "from JSON.", "style": "accent-italic" }
      ]
    }
  ]
}
```

**Fields:**

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `type` | `"card"` | optional | Cards are the default — omitting is fine |
| `template` | string | yes | `"manifesto"`, `"quote"`, `"list"`, or `"minimal"` |
| `theme` | string | yes | Name matching a file in `themes/` |
| `size` | SizeName | yes | One of 14 presets (see below) |
| `blocks` | Block[] | yes (≥1) | Ordered list of content blocks |
| `meta` | object | optional | `title`, `created`, `tags` — used by the studio |

## Deck shape

```json
{
  "type": "deck",
  "defaults": {
    "template": "quote",
    "theme": "noir-crimson",
    "size": "instagram-sq"
  },
  "slides": [
    { "id": "s1", "blocks": [{ "id": "h", "type": "headline", "parts": [{ "text": "Slide 1", "style": "normal" }] }] },
    { "id": "s2", "theme": "terminal-green", "blocks": [{ "id": "h", "type": "headline", "parts": [{ "text": "Slide 2", "style": "normal" }] }] }
  ]
}
```

Each slide inherits from `defaults`; any field on the slide overrides it. Slides require a unique `id`.

## Block types

Seven block types live in `BlockSchema`:

### `headline` — display text
```json
{
  "id": "h",
  "type": "headline",
  "parts": [
    { "text": "Bold statement.", "style": "normal" },
    { "text": " Emphasis here.", "style": "accent-italic" }
  ]
}
```

### `blockquote` — pull quotes with a border
```json
{
  "id": "q",
  "type": "blockquote",
  "parts": [
    { "text": "Short quote.", "style": "bold" }
  ]
}
```

### `text` — paragraph copy
```json
{ "id": "p", "type": "text", "content": "Plain-text paragraph." }
```

### `bullet-list` — labeled rows
```json
{
  "id": "list",
  "type": "bullet-list",
  "items": [
    { "label": "01", "text": "First point" },
    { "label": "02", "text": "Second point" }
  ]
}
```

### `callout` — boxed highlights (same shape as bullet-list)
```json
{
  "id": "call",
  "type": "callout",
  "items": [
    { "label": "Tip", "text": "Keep bullet-list labels under 4 characters for visual balance." }
  ]
}
```

### `divider` — horizontal rule
```json
{ "id": "d", "type": "divider" }
```

### `spacer` — explicit vertical gap
```json
{ "id": "gap", "type": "spacer", "size": "md" }
```

`size` is `"sm" | "md" | "lg"`.

## Part styles

`headline` and `blockquote` blocks hold an array of **parts**, each styled independently:

| Style | Effect |
|-------|--------|
| `normal` | Default weight/color |
| `bold` | Bold weight |
| `italic` | Italic (same color) |
| `accent` | Accent color, normal weight |
| `accent-italic` | Accent color, italic |
| `mono` | Monospace (uses body font family) |
| `muted` | 60% opacity |

This is what makes one line of text read with rhythm: a `normal` start, an `accent-italic` punchline.

| Name | Dimensions | Ratio | Platform |
|------|-----------|-------|----------|
| `twitter` | 1200×675 | 16:9 | Twitter post |

| `facebook-post` | 1200×630 | 1.91:1 | Facebook post / link |

| `facebook-event` | 1920×1080 | 16:9 | Facebook event cover |

| `custom` | user-defined | free | Custom dimensions |

## Validation

Check any card or deck without rendering it:

```bash
bun quoteforge validate content/my-card.json
```

Errors point at the exact path (e.g. `blocks[2].parts[0].style`) and the expected type, which makes template drift obvious before you spend a Puppeteer run on it.
