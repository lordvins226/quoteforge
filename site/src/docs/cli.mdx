QuoteForge ships eight commands. They compose well: `validate` gates CI, `generate` produces one card, `slides` produces a deck, `batch` produces many, `studio` opens the editor, `preview` watches a file, `themes` manages theme JSON, and `doctor` checks the install.

## `generate`

Render a single card JSON to PNG.

```bash
quoteforge generate <file> [options]
```

Options:

| Flag | Default | Description |
|------|---------|-------------|
| `-t, --theme <name>` | (from file) | Override theme |
| `-s, --size <name>` | (from file) | Override size (e.g. `instagram-sq`) |
| `-o, --output <path>` | `outputs/` | Output directory |
| `--scale <n>` | `2` | Pixel ratio (1 or 2) |
| `--no-timestamp` | — | Don't append ISO timestamp to filename |
| `--open` | — | Open the PNG after generation |

## `slides`

Render a deck JSON into numbered PNGs + a ZIP.

```bash
quoteforge slides <file> [options]
```

Options:

| Flag | Default | Description |
|------|---------|-------------|
| `-t, --theme <name>` | — | Override theme for all slides |
| `-s, --size <name>` | — | Override size for all slides |
| `-o, --output <dir>` | `outputs/<deck>/` | Output directory |
| `--slide <n>` | — | Render only slide N (1-indexed), no ZIP |
| `--no-zip` | — | Skip ZIP creation |
| `--no-counter` | — | Disable counter overlay |
| `--concurrency <n>` | `4` | Parallel render workers |
| `--scale <n>` | `2` | Pixel ratio |
| `--open` | — | Open output folder |

## `batch`

Render every JSON file in a directory.

```bash
quoteforge batch <dir> [options]
```

Options include `--theme`, `--size`, `--output`, `--concurrency`, and `--decks` (to also process deck files).

## `validate`

Zod-validate a card, deck, or theme file. Exits non-zero on failure — wire this into CI.

```bash
quoteforge validate content/my-card.json
quoteforge validate --theme themes/custom.json
```

## `preview`

Watch a file and re-render on change. Opens the PNG in your default viewer.

```bash
quoteforge preview content/examples/quote-twain.json
```

## `studio`

Launch the WYSIWYG editor.

```bash
quoteforge studio [file] [--port 4242] [--no-open]
```

## `themes`

Manage theme JSON files.

```bash
quoteforge themes list
quoteforge themes show obsidian
quoteforge themes create my-theme
quoteforge themes duplicate obsidian my-obsidian
quoteforge themes validate themes/my-theme.json
```

## `doctor`

Report install health — bundled assets, user theme directory, resolved Chrome, runtime versions. Exits **1** if Chrome can't be resolved.

```bash
quoteforge doctor
```

## Environment variables

| Variable | Purpose |
|----------|---------|
| `QUOTEFORGE_CHROME` | Absolute path to a Chrome/Chromium/Edge executable. Skips auto-detect. |
| `QUOTEFORGE_INSTALL_DIR` | Used by `install.sh` to pick the install target (default `~/.local/bin`). |
| `QUOTEFORGE_VERSION` | Used by `install.sh` to pin a specific release tag. |
| `XDG_CACHE_HOME` | Overrides `~/.cache` as the base for bundled-asset unpacking and the Chrome cache. |
| `XDG_CONFIG_HOME` | Overrides `~/.config` as the base for user-authored themes. |

## Exit codes

All validation errors exit with code **1**. Render failures exit **2**. File-not-found exits **1**.
