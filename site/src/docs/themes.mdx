A theme is a JSON file in `themes/` that maps named tokens to CSS custom properties. Templates reference these tokens exclusively — **no template hardcodes a color or font**. Swap the JSON, swap the visual identity.

## Theme shape

Every theme validates against the same Zod schema (`src/cli/utils/validator.ts`):

```json
{
  "name": "terminal-green",
  "displayName": "Terminal Green",
  "colors": {
    "background":         "#0B0F14",
    "headline":           "#E5EAF0",
    "accent":             "#22C55E",
    "body":               "#9AA4B2",
    "label":              "#22C55E",
    "blockquote-border":  "#22C55E",
    "blockquote-text":    "#E5EAF0",
    "callout-bg":         "#0F141B",
    "callout-border":     "#1F2A37",
    "bullet-dot":         "#22C55E",
    "slide-counter-bg":   "#00000066",
    "slide-counter-text": "#E5EAF0"
  },
  "typography": {
    "font-headline":     "JetBrains Mono",
    "font-headline-url": "https://fonts.googleapis.com/css2?family=JetBrains+Mono:wght@500;700",
    "font-body":         "IBM Plex Sans",
    "font-body-url":     "https://fonts.googleapis.com/css2?family=IBM+Plex+Sans:wght@400;500;600",
    "headline-size":     "clamp(3rem, 7vw, 5rem)",
    "body-size":         "1rem",
    "line-height":       "1.55"
  },
  "spacing": {
    "padding":   "64px",
    "block-gap": "1.75rem"
  }
}
```

## Field reference

### Top level
| Field | Type | Notes |
|-------|------|-------|
| `name` | string | Stable slug (matches filename) |
| `displayName` | string | Human-readable label (shown in the studio picker) |
| `colors` | object | 12 color tokens (below) |
| `typography` | object | 7 typography tokens (below) |
| `spacing` | object | 2 spacing tokens (below) |

### `colors` (all required; hex strings)
| Token | Role |
|-------|------|
| `background` | Card background |
| `headline` | Headline block text |
| `accent` | `accent` / `accent-italic` parts, bullet dots, borders |
| `body` | Body text, text-block content |
| `label` | Bullet-list labels, callout labels |
| `blockquote-border` | Left bar on blockquote blocks |
| `blockquote-text` | Blockquote text color |
| `callout-bg` | Callout surface fill |
| `callout-border` | Callout outer border |
| `bullet-dot` | Bullet-list marker color |
| `slide-counter-bg` | Deck counter chip background (supports rgba, e.g. `#00000066`) |
| `slide-counter-text` | Deck counter chip text |

### `typography` (all required except `*-url`)
| Token | Role |
|-------|------|
| `font-headline` | Heading family name |
| `font-headline-url` | *(optional)* Google Fonts CSS URL for the heading family |
| `font-body` | Body family name |
| `font-body-url` | *(optional)* Google Fonts CSS URL for the body family |
| `headline-size` | Base headline size (accepts `clamp(...)`, `rem`, `px`) |
| `body-size` | Base body size |
| `line-height` | Body line-height (unitless) |

### `spacing` (both required; CSS length strings)
| Token | Role |
|-------|------|
| `padding` | Card inner padding |
| `block-gap` | Gap between stacked blocks |


## Creating a theme

```bash
quoteforge themes create my-brand
```

Creates `themes/my-brand.json` from a default skeleton that validates against the schema above. Edit the tokens, save, and reference by name in your content:

```json
{ "theme": "my-brand", "template": "quote", "size": "instagram-sq", "blocks": [...] }
```

## Validate before shipping

```bash
quoteforge themes validate themes/my-brand.json
```

Catches malformed hex codes, missing required tokens, and typos in font-URL query strings before they reach Puppeteer.

## Common mistakes

- **Using `background-color` instead of `background`.** The token is named `background` to match the CSS property, but the JSON key drops the suffix.
- **Nesting `fonts` or using `label` at the top level.** Earlier drafts used those names; the real schema uses `typography` and `displayName`.
- **Setting `padding: 64` without a unit.** All spacing tokens are full CSS length strings — include `px`, `rem`, or `%`.
